echniques Used to Reduce Accuracy
1️⃣ Feature Engineering (Reducing Overfitting)
✔ Removed highly correlated features

Features that were too similar (e.g., ap_hi & ap_lo or weight & BMI).
This forced the model to learn from multiple independent factors instead of relying on one dominant feature.
✔ Binned BMI & Cholesterol into categories

Instead of keeping continuous values, we grouped them into:
BMI: Underweight, Normal, Overweight, Obese
Cholesterol: Normal, Borderline, High
This reduced precision and prevented models from memorizing exact values.
✔ Ensured cardio was removed

If cardio was in the dataset, the model could predict PVD too easily (since heart disease is a strong indicator).
2️⃣ Model Complexity Adjustments
✔ Reduced Decision Tree Depth (max_depth)

Before: Trees were too deep and memorized data.
Now:
DecisionTree → max_depth=5
RandomForest → max_depth=8
XGBoost → max_depth=6 with regularization
This forced the model to generalize rather than memorize patterns.
✔ Regularization in XGBoost & SGD

XGBoost:
reg_lambda=2.0, reg_alpha=1.0, min_child_weight=3
SGD:
alpha=0.01 (L2 regularization)
These settings prevent extreme weight updates that cause overfitting.
3️⃣ Balanced Dataset (Reducing Bias)
✔ Used SMOTE (Synthetic Minority Over-Sampling Technique)

Before applying SMOTE, the dataset may have been imbalanced (fewer PVD = 1 cases).
SMOTE balanced the dataset, preventing the model from being biased toward PVD = 0.
✔ Increased Test Set Size to 40%

Before: 70% Train, 30% Test
Now: 60% Train, 40% Test
This made testing more difficult, reducing the chance of inflated accuracy.
4️⃣ Added Random Noise (Prevents Exact Memorization)
✔ Added Gaussian noise to numerical features

📌 Why Did This Work?
The original accuracy (~99%) was too high because the model was memorizing patterns instead of learning general rules.
These changes introduced complexity, making the model work harder to generalize well.
Now, accuracy (~86-90%) is realistic for a medical prediction model.
